Ball Project

This work consists of developing a simple graphical application based on the event-driven programming paradigm. The objective is to represent a ball inside a canvas and allow the user to move it in different directions through button interactions.

The interface displays a canvas where the ball is drawn, along with four control buttons: Up, Down, Left, and Right. Each time the user clicks one of these buttons, an event is triggered that updates the ball's position on the canvas in the selected direction.

If you want to see the original code click here.

In this second version, since it is a project meant for experimentation, the original exercise was expanded to make it more complex and interactive. The goal shifted from moving the ball through simple fixed displacements to simulating the motion of a ball in a 2D physical environment.

For this purpose, collisions with the canvas borders were added along with basic forces such as gravity and air resistance. In this way, the buttons no longer move the ball directly but instead apply a force in the selected direction, resulting in more realistic motion with acceleration, slowing down, and bouncing when hitting the walls.

Poker Project

This project originated as one of the final assignments in my university Web Development course. The original code can be found here.

The main objectives were to practice development based on the MVC (Model-View-Controller) pattern, event-driven programming, graphical programming, and the use of object-oriented programming.

The initial result was a simple application that simulated a poker game between two five-card hands. From that foundation, I decided to keep working on it to make the project more realistic and complex, while continuing to strengthen my skills.

Among the improvements I introduced are a third canvas representing the community cards on the table, emulating the dynamics of a real poker game, as well as a betting system that allows the player to choose an amount and face an AI-controlled opponent capable of deciding whether to call or fold.

Overall, the project combines the academic principles of the original assignment with a more ambitious development that enhances both the game logic and the interactive experience.
